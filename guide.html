General

If it is given to you as a Word Document, and it needs to be uploaded, chances are you should convert it to PDF format first.

If it is given to you as a PDF Document, check if it has been prepared by pressing Ctrl-D. If it isn't, see below and Ctrl-S (save) when done.

Here is a reference sheet that will help you on this matter. Save it for quick access.

Metadata

Change the Title and Author. Usually Author will be AGF Investments Inc. for English documents and Placements AGF Inc. for French documents.

Security

This one, the second tab, is important. This ensures our PDFs cannot have its contents copy-pasted and is a small security safeguard that is part of policy.

  1. Check "Restrict Editing..." and the shaded out section will become available.
  2. Printing method should be "HIGH RESOLUTION"
  3. The Permissions password is "agfagf"
  4. The Compatibility should be changed to "Acrobat 6.0 and later".

Initial View

The fourth tab. Just make sure the layout is "Single Page" and the Magnification is set to "Fit Width".

Starting Request

When you first access the eMarketing Tasks page, there are certain tasks assigned to "DAP eMarketing" that have not been started yet. They should be completed in order of priority (see due date).

Click on the "Task Name" and edit task. Assign to yourself, and set the "Task Status" as "Started". This will indicate to other people that you have started this task (one task does not need the same person doing it twice!). This just for good communication.

Ready for Review

Once a task has been completed, you must ask for it to be approved.

Assign to the recipient. Set task status as "Ready for Review". Remember to insert all relevant links i.e where did you post it? If it is on the agf.com or agfiq site, you should send the demo links as the full site would not have the changes yet. Also a direct link to the file uploaded would be needed. As soon as you submit, an email will be sent out.

Logging Progress

Usually tasks require a little bit of back and forth via "Return"ed SMARTrequests. However, as soon as a task has been approved (and published), you should log it in the Week in Review.

  1. Add new item.
  2. Make a title (usually name of document or SMARTrequest)
  3. Paste the link of the document (i.e. pdf) in the appropriate place i.e. Investor if on agf.com
  4. Save

Notes:

  • If there are lots of separate documents, list each separately.
  • Do not paste the webqa/demo links i.e Only do this once the task has been approved and published.
  • Do not paste the index link.

Completing Request

Only once a task has been approved and logged can you remove it from the queue. This is because it will be much more difficult to access this request after doing this step.

  1. Assign to yourself.
  2. Set Task status to "Complete". Save. You will be taken back to eMarketing tasks, sans the task you just completed.

Switching Sites

You will probably need to switch sites first before doing anything. This can be done by accessing the dropdown menu at the top left corner of Cascade. You will most likely be in "AGF Files" for most of your tasks.

Uploading Files

Sometimes you'll need to upload a file like a PDF or an image file to Cascade

Navigate to the folder you are uploading the file or document to. There's probably a similar file in the folder, so copy it and click on "Edit". Edit anything you need to (title, summary, date, etc.) and keep in mind of the naming conventions from the other files in the folder. Drag the file that you're uploading to the bottom of the content tab where it says "Drop a file here". Hit submit, uncheck "use workflow", and publish the file.

Replacing Files

You may need to replace a PDF, image file, or other files in Cascade.

Navigate to the folder containing the file you're replacing. Open the file you're replacing and click on the "Edit" button on the top right. You may or may not need to edit the title or any other information depending on the request. Drag the new file to the bottom of the content tab. Submit, uncheck "use workflow", and publish the file. If the file is being used somewhere on the site, click on "More" and go to relationships. Click on the pages under relationships and publish them to update them with the new file.

Editing Text

You will often need to edit some text for the AGF website such as an article or a module on the site.

First find the page you will be working on through Cascade. The search bar at the top right of Cascade can be handy to find the page. Once you're at the page, click on "Edit". Scroll down a bit and you'll see some items. Find the item containing the text you'll be editing and click on "Source Code" (the icon that looks like "<>") to edit the text. Be careful with the formatting if you're not familiar with HTML. Replace or change the text inside (it'll probably be between the "<p></p>" for content and "<h1></h1>" or something similar for headings). Submit, uncheck "use workflow" and publish the site.

Adding an Update

Once you're at the Week in Review page, click on "Document Updates Week in Review" unless you have been specifically instructed to create an item in the "Webpage Updates Week in Review" page. Click "new item" near the top of the page.

Inputting the Info

Fill in the information as needed, including whether it is a new document or an update to one. The fields after the date field are where you insert the relevant links. You will mainly be pasting Smartlink links so focus on SMARTLink EN and SMARTLink FR. Hit save and you're done!

Exporting to Excel

First off, when doing Product 411, make sure you use Internet Explorer and not Chrome. Open both the Webpage Week in Review and the Document Updates Week in Review in IE.

Click "export" and then click "List" on the top left corner. Open/Save the Excel file and click "Enable" at the security notice. Copy the first column(column A) and paste it into notepad. Save the notepad as a .html file by putting ".html" instead of ".txt" at the end of the name when saving the file. Do this for both types of Week in Reviews.

Inserting into Outlook

Make a new email in Outlook (we won't be sending the email it's just for formatting). Click "Attach File", select one of the Week in Reviews, click the ▼ beside “Insert”, and then select “Insert as text”.

Formatting in Outlook

Click on the table you just inserted and click on "design" at the top and select the specified layout. Now the table should be fully formatted. Now insert the other Week in Review into the same email and repeat the steps above.

Posting

Copy the formatted tables and go to the Product 411 page. Click "Create a post" and paste the tables by clicking the under ▼ "Paste" and selecting "Paste". Now copy the titles provided below and put them above the corresponding table. Don't forget to fill in the dates! Copy these titles:
AGF.com, AGFAdvisor.com, AGF.com/Institutional, SMARTLink Webpage Week In Review ([MONTH] DD, YYYY – [MONTH] DD, YYYY)
Document updates Week In Review ([MONTH] DD, YYYY – [MONTH] DD, YYYY)

Leave the summary blank. In the categories section, select "Week in Review", click "Add", and change the status to "Published". Double check everything and hit the publish button when you're ready.
